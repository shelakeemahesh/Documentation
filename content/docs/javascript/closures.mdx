# Closures

A closure is the combination of a function bundled together with references to its surrounding state (the lexical environment). In JavaScript, closures are created dynamically every time a function is defined, allowing inner functions to access outer variables even after the outer function has finished executing.

## Core Concepts

Understanding closures is key to encapsulating data state and managing function memory.

### Lexical Scope Retention
When a function returns another function, the returned function retains a reference to the variables in its parent's scope. The JavaScript garbage collection engine does not remove these parent variables from memory because they are still referenced by the child function.

### Data Encapsulation & Private Variables
Closures allow you to emulate private variables (like private fields in object-oriented programming). You can restrict direct access to a variable and only allow updates through public getter and setter functions.

---

## Code Example

Here is an implementation of a counter closure that encapsulates its state:

```javascript
function createCounter() {
    // Private state variable
    let count = 0;
    
    return {
        increment() {
            count++;
            return count;
        },
        decrement() {
            count--;
            return count;
        },
        getCount() {
            return count;
        }
    };
}

const counter = createCounter();
console.log(counter.increment()); // 1
console.log(counter.increment()); // 2
console.log(counter.getCount());  // 2
// Direct access to count is impossible (counter.count is undefined)
```

## Best Practices

*   **Use for Data Hiding**: Protect sensitive variables by wrapping them in closures and exposing only specific controls.
*   **Release Memory References**: Set closure-retaining variables to `null` when they are no longer needed to allow the garbage collector to free memory.
*   **Avoid Closures inside Loops**: Avoid creating functions inside loops, as this allocates multiple closure scopes and increases memory usage.
*   **Leverage for Factory Functions**: Use closures to create customized configurations based on initial parameters.

---

## Common Mistakes

*   **Creating Memory Leaks**: Retaining references to large DOM structures inside closures, preventing the browser from garbage collecting them.
*   **Expecting shared variables to decouple**: Assuming multiple closure instances share the same state, when each call to the outer function creates a new, independent closure scope.
*   **Confusing closure references with static values**: Modifying parent variables in closures and expecting prior outputs to remain static.

<Callout type="tip" title="Practical Use Case">
  Closures are widely used in functional programming patterns, such as currying, memoization, and event handler configuration.
</Callout>

---

## Key Takeaways

*   Closures allow functions to access outer scope variables even after the outer function returns.
*   Use closures to create private variables and encapsulate data state.
*   Each call to the outer function creates a new, independent closure scope.
*   Clean up closure references when they are no longer needed to prevent memory leaks.
