# Flexbox

Flexbox (Flexible Box Layout) is a one-dimensional CSS layout model designed to distribute space and align items along a single axis (either horizontally in rows or vertically in columns), even when element sizes are dynamic or unknown.

## Core Concepts

Understanding Flexbox axes and alignment properties is key to building responsive, dynamic UI layouts.

### Main Axis vs. Cross Axis
-   **Main Axis**: The primary axis along which flex items are laid out. Controlled by `flex-direction` (defaults to horizontal `row`).
-   **Cross Axis**: The perpendicular axis. If the main axis is a `row`, the cross axis is a vertical `column`.

### Primary Flex Container Properties
-   **`display: flex`**: Turns an element into a flex container and its children into flex items.
-   **`justify-content`**: Aligns items along the **Main Axis** (e.g. `center`, `flex-start`, `space-between`).
-   **`align-items`**: Aligns items along the **Cross Axis** (e.g. `center`, `stretch`, `flex-end`).
-   **`flex-wrap`**: Controls whether flex items wrap onto multiple lines when container space overflows.

---

## Code Example

Here is an implementation of a responsive navbar layout using Flexbox:

```css
.navbar-container {
    display: flex;
    justify-content: space-between; /* Push logo and links to opposite ends */
    align-items: center;            /* Center links vertically */
    padding: 16px 24px;
    background-color: #09090b;
    border-bottom: 1px solid #27272a;
}

.nav-links {
    display: flex;
    gap: 16px;                      /* Easy space between sibling links */
    list-style: none;
    margin: 0;
    padding: 0;
}
```

## Best Practices

*   **Use gap for Spacing**: Use the `gap` property to add space between flex items, avoiding the need for helper margin overrides.
*   **Keep Layouts One-Dimensional**: Use Flexbox for single-axis layouts (like headers, button groups, or sidebars). For two-dimensional grid layouts, use CSS Grid.
*   **Avoid Absolute Widths on Flex Items**: Allow elements to size dynamically using flex parameters rather than hardcoding absolute `width` values.
*   **Use auto Margins to Push Items**: Use `margin-left: auto` on a flex item to push it and all subsequent items to the far right of the container.

---

## Common Mistakes

*   **Confusing Main and Cross axes properties**: Attempting to use `justify-content` to align elements vertically on a horizontal row layout.
*   **Forgetting flex-wrap**: Leaving `flex-wrap: nowrap` active on responsive lists, causing items to overflow the container boundaries on small screens.
*   **Using absolute pixel margins for layout alignment**: Hardcoding margins instead of leveraging Flexbox alignment properties (`justify-content`, `align-items`).

<Callout type="tip" title="Centering Anything">
  To center an element both horizontally and vertically, wrap it in a flex container and apply: `display: flex; justify-content: center; align-items: center;`.
</Callout>

---

## Key Takeaways

*   Flexbox is built for one-dimensional row or column layouts.
*   `justify-content` controls main axis space distribution; `align-items` controls cross axis alignment.
*   Use `gap` instead of vertical/horizontal margins to add space between flex items.
*   Use `flex-wrap` to support multi-line wrap grids.
