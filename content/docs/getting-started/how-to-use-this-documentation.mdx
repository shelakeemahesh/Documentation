# How to Use This Documentation

This documentation hub is designed to be an interactive, searchable reference guide for software engineers. Every page includes conceptual foundations, runnable code examples, best practices, common pitfalls, and architectural system designs to provide deep, practical knowledge.

## Core Concepts

Getting the most out of this portal requires understanding how its components and sections are structured.

### Interactive Search and Hotkeys
The site features client-side Orama search. You can press \`⌘ K\` or \`Ctrl K\` at any time to open the global search palette and instantly search across all topic files, headings, and code snippets.

### Document structure
*   **Regular Pages**: Explain specific technical APIs or syntax, including code blocks, best practices, and common mistakes.
*   **System Design Pages**: Address scale, requirements, high-level architectures (using Mermaid.js diagrams), and real-world system examples.

## Best Practices

*   **Use the Search Bar**: Leverage the Cmd+K search palette to quickly find code signatures when troubleshooting bugs.
*   **Inspect the Diagrams**: Study the Mermaid diagrams on the System Design pages to understand component dependencies.
*   **Verify Code Locally**: Clone the code repositories and run them using the local dev servers listed in each section.
*   **Check the Callouts**: Pay close attention to Warning and Danger callouts to avoid common configuration bugs.

---

## Common Mistakes

*   **Bypassing the code examples**: Reading the documentation like a book without executing the accompanying code snippets locally.
*   **Overlooking the sidebar structure**: Missing related topics because of not utilizing the collapsible sidebar categories.
*   **Ignoring local terminal errors**: Continuing to code when the local Vite or Spring dev server is throwing compile warnings.
